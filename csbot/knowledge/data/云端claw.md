# 云端部署 Kimi Claw 知识库

&gt; 适用于：通过 Kimi 官网一键部署的 Kimi Claw 用户（非本地/自行部署）

---

## 一、基础认知

### Q1: 什么是云端部署的 Kimi Claw？
**A:** Kimi 官方提供的一键部署服务。用户无需购买服务器或命令行配置，Kimi 自动将 OpenClaw 部署到云端，并自动配置：
- Kimi K2.6 Thinking 模型
- Kimi Code 会员权益额度
- Kimi Web Search 联网搜索服务

**前置条件：** 仅支持 Allegretto 及以上会员计划。

---

### Q2: 如何创建云端 Kimi Claw？
**A:** 
1. 访问 https://www.kimi.com/bot
2. 点击【创建】，等待自动完成配置
3. 修改 Bot 昵称
4. 点击【设置】→【聊天频道】，扫码配置微信/飞书/企业微信

### Q3: 云端部署和本地部署有什么区别？
**A:**

| 维度 | 云端部署 | 本地/关联已有 OpenClaw |
|------|----------|------------------------|
| 部署方式 | 一键自动，无需服务器 | 需自行准备设备，安装插件 |
| 配置复杂度 | 零配置，自动关联 API | 需手动配置模型、搜索等 |
| 适用会员 | Allegretto 及以上 | 无限制 |
| 维护责任 | Kimi 官方维护 | 用户自行维护 |
| 升级方式 | 官方统一推送 | 用户手动操作（风险高） |
---

## 二、平台接入与聊天频道

### Q4: 云端 Kimi Claw 支持哪些聊天平台？
**A:** 微信、飞书、企业微信、微博、钉钉（以当期控制台展示为准）。

**注意：** 微信插件仅限 2026 年 3 月 22 日后新创建的实例内置，旧实例需升级或重建。

---

### Q5: 为什么我的 Kimi Claw 看不到"一键扫描关联微信/飞书"的选项？
**A:** 原因：企微/微信插件依赖新版本 OpenClaw，3 月 22 日之前创建的实例不支持。

**解决方案（二选一）：**
1. **保留文件升级：** 设置 → 【恢复初始版本】→ 保留工作空间升级 → 重新配置聊天软件
2. **不保留文件：** 删除 bot 后新建
3. **等待官方推送：** 无损升级预计下个版本支持，关注页面内升级提示

---

### Q6: 微信消息为什么没有同步到 Kimi 网页端？
**A:** 这是 OpenClaw 的官方设计。微信消息属于微信渠道频道，聊天记录只在微信内可见。同理，飞书/企微等其他 IM 的信息也不会在 Kimi 显示。

---

### Q7: 飞书群里 @机器人没有反应怎么办？
**A:** 
飞书机器人默认不响应群内对话。请按《平台接入与鉴权》中的飞书章节执行配置并重启 Kimi Claw。

---

## 三、升级与版本管理

### Q8: 云端 Kimi Claw 可以手动升级 OpenClaw 版本吗？
**A:** **绝对不可以。** 手动升级会导致：
- Kimi 插件失效
- 微信、飞书、企微、微博等插件不可用
- 配置与数据覆盖风险（密钥丢失、定时任务清空、Skill 失效）

**正确做法：** 等待官方统一推送升级。

---

### Q9: 误操作升级导致插件失效怎么办？
**A:** 在设置中选择【恢复初始设置】恢复到可用版本。
- 工作空间和记忆会保留
- 需要重新配置聊天机器人

---

### Q10: 可以手动升级飞书插件吗？
**A:** **不建议。** 飞书插件升级会同时升级 OpenClaw 版本，导致与当前环境不兼容。新创建的 Kimi Claw 已内置适配的最新版本插件。

---

## 四、故障排查与修复

### Q11: Kimi Claw 不回消息或回复很慢怎么办？
**A:** 按顺序尝试：

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 再发一条消息或刷新页面 | 排除临时卡顿 |
| 2 | 设置内重启 Gateway 服务 | Gateway 是消息收发中枢 |
| 3 | 点击【修复 Kimi Claw 配置】 | 自动修复连接问题 |
| 4 | 选择【恢复初始配置】 | 保留工作空间/记忆/定时任务，重置运行环境 |

---

### Q12: 提示"与云端电脑的连接已断开"怎么办？
**A:** 
- **连接稳定性问题：** 更新 Kimi Claw 插件和软件版本
  - Web 端：升级插件
  - Android：升级至 ≥ v2.6.3
  - iOS：升级至 ≥ v2.6.1
- **配置损坏问题：** 先尝试【修复 Kimi Claw 配置】，无效则【恢复初始设置】
- **仍无法解决：** 请联系人工客服进一步处理，提供 Bot ID 以便排查

---

### Q13: 提示"与 Kimi 连接已经断开"怎么办？
**A:** 可能是 AI 在修改配置过程中改错了 OpenClaw 配置。
- **当前方案：** 删除 bot 重建（会丢失工作空间内容）
- **即将支持：** 保留 Workspace 内容修复配置的功能（开发中）

---

### Q14: 配置了 Tailscale 后 Kimi Claw 断开连接怎么办？
**A:** Tailscale 覆盖了系统 DNS 配置。修复步骤：
1. 打开 [TailScale DNS 配置页面](https://login.tailscale.com/admin/dns)
2. Global nameservers 处添加 Google Public DNS 和 Cloudflare Public DNS
3. 打开右上角 Override DNS Servers 按钮
4. 等待约 30 秒恢复

## 七、命令与终端

### Q19: 云端 Kimi Claw 可以使用命令行吗？
**A:** 可以。两种方式：
1. 【设置】→【打开终端】直接输入指令
2. 将需要执行的命令告诉 Bot，它会帮你执行

**注意：** Terminal 和插件共享交互通路，重启 openclaw gateway 会导致 terminal 断开连接。

---

### Q20: 如何查看常用命令？
**A:** 在 Claw 对话框输入 `/help` 查看所有可用命令。常用命令包括：

| 命令 | 功能 |
|------|------|
| `/status` | 系统运行状态 |
| `/new` | 开启新 Session |
| `/reset` | 重置当前 Session |
| `/skills` | 列出已安装 Skill |
| `/cron` | 查看/管理定时任务 |
| `/memory` | 查看记忆条目 |
| `/config` | 查看/修改配置 |
| `/reasoning off` | 隐藏思考过程（仅隐藏展示，不影响模型思考） |

## 九、飞书诊断专用

### Q23: 飞书连接问题如何自助诊断？
**A:** 向 Kimi Claw 发送：
- `/feishu start` — 确认安装状态
- `/feishu doctor` — 检查配置是否正常
- `/feishu auth` — 批量完成用户授权
