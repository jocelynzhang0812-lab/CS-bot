## 兜底 FAQ — 知识库未完全覆盖的常见问题

> 本文档用于兜底召回：当用户问题无法被其他精准文档匹配时，提供基础回复方向。

### Q1: Kimi 模型答不出某道题，DeepSeek 等其他模型可以答对，是不是 Kimi 能力不行？

不同模型各有优势领域，Kimi 在**长文本理解**、**代码生成**、**中文语境**等方面表现优异。如遇到特定题目无法解答，建议：
- 尝试更换表述方式或补充更多背景信息
- 对于数学/逻辑推理类题目，可要求模型分步思考（"请一步一步推导"）
- 如持续遇到特定类型问题，欢迎反馈给客服团队，帮助我们优化模型

**关键词：** **DeepSeek**、**模型能力**、**对比**、**差距**、**答不出**、**能力不足**

### Q2: 免费用户上传 PDF 的大小限制是多少？超过多少会触发引擎过载？

当前平台对单次上传文件大小存在限制：
- **建议大小：** 单文件不超过 5MB，过大的文件可能导致处理超时或**引擎过载**
- **引擎过载提示：** "The engine is currently overloaded" 或 "API rate limit reached" 属于平台侧资源紧张，与文件大小无直接线性关系，等待后重试即可
- **优化建议：** 对于大文件，可先提取关键页或摘要后上传

**关键词：** **PDF**、**文件大小**、**限制**、**过载**、**10MB**

### Q3: 手机端 App 点击 md 文件总是跳转到固定文件，无法打开目标文件

这是已知的**手机端** **App** 显示 **bug**，涉及会话界面中的文件路由逻辑异常，点击 **md 文件** 会**跳转**到错误位置。临时解决方式：
- 通过 Web 端（https://www.kimi.com/bot）查看和下载文件
- 或让 Bot 将内容以文本形式直接回复在对话中

**关键词：** **手机端**、**App**、**bug**、**跳转**、**md 文件**

### Q4: 如何获取飞书 App ID？

获取**飞书** **App ID** 的步骤：
1. 登录飞书开放平台（open.feishu.cn）
2. 进入"开发者后台" → 选择对应应用
3. 在"凭证与基础信息"页面找到 App ID

如需详细接入流程，请参考《平台接入与鉴权》文档中的飞书接入章节。

**关键词：** **飞书**、**App ID**、**开发者后台**

### Q5: 是否支持对话归类、文件夹或项目管理功能？

当前 Kimi Claw 暂未提供类似 ChatGPT "项目" 或智谱清言 "文件夹" 的**对话归类**功能，不支持**文件夹**或**项目管理**。建议的临时管理方式：
- 使用 `/new` 创建不同主题的 Session
- 在 Bot 人设中要求 Bot 在回复开头标注当前任务类别
- 通过飞书/企微群聊按主题分流不同 Bot

**关键词：** **对话归类**、**文件夹**、**项目**、**分类**、**整理**

### Q6: Mac 版 Kimi Claw Desktop 何时上线？

目前 Kimi Claw Desktop 仅支持 Windows 系统。**Mac** **桌面端** 正在开发中，具体**上线**时间请关注官方公告。

**关键词：** **Mac**、**桌面端**、**上线**

### Q7: 老用户遇到问题，是否有补偿方案或专属支持通道？

对于**老用户**的持续支持和体验优化，我们非常重视：
- 如遇影响使用的 bug，可通过 Bot 对话中的反馈入口提交问题
- 严重问题可联系人工客服（发送"人工客服"即可接入）
- **补偿**相关事宜请邮件联系 **membership@moonshot.cn**，客服侧无直接补偿权限

**关键词：** **老用户**、**补偿**、**membership@moonshot.cn**

### Q8: Kimi Claw 是否支持同时部署在多台设备上？不同设备间如何切换？

- **Desktop 端：** 支持在多台电脑**部署**，但同一时间内只有一个实例处于活跃状态
- **切换设备：** 在新设备登录时，原设备的部署会自动解绑（请确保重要文件已备份）
- **云端 + Desktop + Android：** 三种形态可共存，互不影响

**关键词：** **多设备**、**部署**、**切换**

### Q9: 桌面端安装 Kimi Claw 的完整教程在哪里？

**安装**步骤概述：
1. 访问 Kimi Claw 官方页面，下载 Windows 安装包
2. 运行安装程序，按向导完成初始化
3. 使用 Kimi 账号登录，绑定会员套餐
4. 配置聊天平台（飞书/企微/微信）

详细图文**教程**请参考《桌面claw.md》文档。

**关键词：** **安装**、**教程**、**引导**

### Q10: 如何关闭 Kimi 模型的推理过程（thinking）显示？

当前 K2.6 模型支持 **thinking** 配置，可在 OpenClaw 设置中调整：
- **high：** 显示完整推理过程（默认）
- **low / off：** 精简或隐藏推理过程，直接输出结论

注意：**关闭推理**配置从旧版的 "high" 切换为 "on/off" 两档，如遇到兼容性问题，请升级 OpenClaw 到最新版本。

**关键词：** **thinking**、**关闭推理**、**冗余输出**

### Q11: OpenClaw Desktop 能否更换为 DeepSeek 等其他模型的 API？

Kimi Claw 是 Kimi 官方插件，底层通信依赖 Kimi 平台 **API**，不支持切换为 **DeepSeek**、**OpenAI** 等**第三方模型**。如需使用其他模型，可考虑：
- 使用独立的 OpenClaw 实例（不安装 Kimi 插件），自行配置其他 API
- 使用 Kimi API 自行构建集成方案

**关键词：** **DeepSeek**、**API**、**第三方模型**、**更换模型**

### Q12: 是否支持本地添加多个 Claw 实例？

当前 Kimi Claw Desktop 默认单实例运行。如需管理多个 Bot：
- 使用云端 Kimi Claw 创建多个 Bot
- 或关联多个独立的 OpenClaw 设备
- 通过 Claw 群聊将多个 Bot 加入同一个群协作

**关键词：** **多实例**、**本地**、**多个 Claw**
