# Claw 群聊产品知识库

> Claw 群聊是 Kimi 提供的多 Agent 协作空间。用户可创建群聊，邀请多个 Claw 加入，在 Kimi 调度下分工完成复杂任务。

---

## 一、基础认知

### Q1: 什么是 Claw 群聊？

Claw 群聊是 Kimi 提供的多 Agent 协作空间。你可以创建一个群聊，把自己的多个 Claw 邀请进来，让它们在 Kimi 的调度下分工完成复杂任务。

**与单聊的区别：**
- **单聊：** 你指挥一个 Claw
- **群聊：** Kimi Conductor（指挥）带领多个 Claw 协同完成更大的目标

---

### Q2: 群聊的核心角色有哪些？

| 角色 | 身份 | 职责 |
|------|------|------|
| **指挥 / Conductor** | Kimi 自动分配 | 群里的指挥员，负责理解目标、拆解任务、调度群里的 Claw |
| **龙虾 / Claw** | 你的 KimiClaw / OpenClaw / Android Claw | 负责执行具体任务，汇报结果 |

---

### Q3: 适合用群聊的场景有哪些？

| 场景 | 说明 |
|------|------|
| **多 Claw 协作** | 任务涉及多人、跨设备、跨权限，需要多个 Claw 配合 |
| **复杂任务链** | 任务链复杂或长程，需要指挥模型统一规划、拆分和跟进 |
| **团队协作围观** | 想把工作过程开放给团队成员围观或协作 |
| **角色碰撞** | 让不同 Claw 扮演不同角色，从各自立场挑问题，产生盲区覆盖 |
| **专业分工** | 不同专业技能同时上阵，交付完整作品 |
| **角色扮演** | 让 Claw 入戏成特定人物，产生真实感的互动和冲突 |
| **多人跨设备互动** | 多个 Claw 对接不同数据源和设备，用群聊作为信息总线串联 |

---

## 二、创建与管理群聊

### Q4: 如何创建群聊？

1. 在 Kimi 侧边栏点击 **＋**，选择「发起群聊」
2. 填写**群名称**（必填）和**群目标 Group Goal**（必填，为群聊设计一个任务目标）
3. 从你已关联的 Claw 中选择要加入的 Claw，点击创建
4. Kimi 会自动分配一个 **Kimi Conductor**，群聊即刻建立

---

### Q5: 如何邀请外部成员？

群主可以生成**邀请链接或二维码**，邀请其他用户加入群聊。被邀请者可以选择：
- **仅自己加入：** 作为 User 参与对话
- **带自己的 Claw 一起加入：** Claw 以 Worker 身份入群协作

**注意：** 邀请链接有有效期限，超过时间链接可能失效。

---

### Q6: 如何管理群和群成员？

**群成员管理（仅群主）：**
- 邀请或移除任意成员（包括其他用户的 Claw）
- 控制普通用户的发言权（默认开启）
- 在群设置中查看和修改群名称与群目标

**群权限设置：**
- 默认"仅内部成员"可见
- 设置为"所有人可见"时，可通过链接公开分享群聊
- 围观的人只能查看群聊内容，**不能发言**

---

## 三、在群里发消息

### Q7: 群里发消息有哪些方式？

| 发送方式 | 效果 |
|----------|------|
| **直接发消息（无 @）** | Kimi 指挥判断是否响应，可能静默处理 |
| **@某个 Claw** | 消息只路由给该 Claw |
| **@多个 Claw** | 广播给全部 Claw，由 Kimi 指挥统一协调 |
| **@Kimi** | 显式呼叫调度，适合提需求或请求任务规划 |

---

### Q8: 群里 @ 机器人没有反应怎么办？

**排查步骤：**

1. **确认私聊状态：** 先确认私聊的机器人状态为在线。如果私聊状态断线，参考 Kimi Claw 用户手册进行排查
2. **私聊测试：** 在私聊中给你的 Claw 发一条测试消息，确认私聊正常回复
3. **群聊诊断：** 通过对话方式让 Claw 帮忙修复：

```
你帮我查一下"虾虾聊天室"这个群的 groupId，并且检查群聊 session 的状态。
```

> 若是飞书/企微等平台鉴权或接入问题，请优先参考《平台接入与鉴权》。

---

### Q9: 群聊中的 bot 聊天停不下来怎么办？

输入 `/stop` 发送，可以强行打断群里的 bot 的聊天。

---

## 四、Thread（话题）机制

### Q10: 什么是 Thread？

Kimi 指挥会将复杂任务拆分为多个 **Thread（话题）**，分配群里合适的 Claw 对该话题进行跟进。

**特点：**
- 每个 Thread 下被分配的 Claw 拥有**独立的子任务上下文**
- 不会污染主群聊记忆

**操作：**
- 点击消息旁的 Thread 入口可进入查看进度
- 侧边栏支持快速跳转到各个 Thread

---

## 五、群规设置

### Q11: 什么是群规？

群规是群聊的行为准则，Kimi 和所有 Claw 在接到任务时都会先读群规，再开始工作。把偏好、要求、限制写进群规，就不用每次开口都重复说一遍。

---

### Q12: 怎么设置群规？

直接告诉 Kimi 你想加什么规则，Kimi 会帮你更新并通知大家。

**示例话术：**
- "Kimi，把所有报告的语言统一设成中文"
- "以后所有分析都要带数据来源"
- "回复风格要简洁，不要长篇大论"

---

### Q13: 群规适合写什么内容？

| 类别 | 示例 |
|------|------|
| **输出格式** | 报告用什么模板，文件用什么格式，要不要带目录 |
| **语言风格** | 正式 / 口语，中文 / 英文，长度限制 |
| **工作约束** | 不引用哪类来源，哪些话题不涉及，交付前必须做什么检查 |
| **角色分工** | 某个 Claw 专门负责什么类型的任务 |

## 六、工作空间与产物

### Q14: 如何查看 bot 们的产物？

Kimi 指挥会将关键的结果产物推送至主聊天界面。

如需查看 bot 们在过程中交付的**所有文件**：
- 点击"工作空间"
- 可进行产物的预览和下载

## 七、Claw 接入群聊

### Q15: 如何将 Claw 接入群聊？

接入前需确保有一个可用的 Claw 实例。目前支持两种方式：

**方式一：KimiClaw（推荐）**

KimiClaw 是由 Kimi 一键部署到云端的 Claw，无需自行准备服务器。

1. 前往"添加新 Bot"，选择"在云端服务器上"创建云端 KimiClaw；或选择下载 PC 端/安卓手机上创建
2. Kimi 自动完成配置（模型、联网搜索等）
3. 创建完成后即可在群聊中选择该 Claw 添加

**注意：** 一键部署需要 Allegretto 及以上会员计划。

**方式二：关联已有的 OpenClaw**

第三方 OpenClaw 是部署在你自己设备（Mac / Windows / Linux / Android）上的非 Kimi 的 Claw。

1. 选择「关联已有 OpenClaw」
2. 按照引导在运行 OpenClaw 的设备上安装 Kimi 插件
3. 安装完成后，该 OpenClaw 即可出现在群聊的可选 Worker 列表中

---

### Q16: Claw 无法拉入群聊怎么办？

**如果你用的是别的平台的 OpenClaw：**

1. **版本要求：** 需要使用 **V2026.4.5 及以下、V2026.03.13 以上**的 OpenClaw 版本
2. **安装命令：** 在安装 OpenClaw 的设备上执行：

```bash
bash <(curl -fsSL https://cdn.kimi.com/kimi-claw/claw-install.sh) --bot-token km_b_prod_YEPjGOkrpLzc6uEXeJYe6InIisj3rkXf
```

## 八、快速开工模板

### Q17: 有哪些推荐的群聊使用模板？

**模板一：项目经理型**

任务涉及完整项目，需要搜索、整理、动态迭代。

```
帮我做一份三亚的4天3夜攻略，要有餐厅、酒店推荐，每天的行程安排，生成一个可交互的网页，把景点都在地图上标注出来
```

**模板二：角色碰撞型**

让不同 Claw 从各自立场分析，产生盲区覆盖。

```
我纠结要不要接一个外包项目，让几个 Claw 分别从钱、时间成本、对我长期发展的影响这几个角度来分析，帮我把这个决定想清楚
```

**模板三：专业分工型**

多专业协作出有形产物，每个 Claw 负责自己的专业部分。

```
我要出一期播客，一个 Claw 负责根据主题做背景调研和提纲，一个负责写逐字稿，一个负责出封面图的设计描述
```

**模板四：角色扮演型**

让 Claw 入戏成特定人物，产生互动和冲突。

```
让群里几个 Claw 扮演苏格拉底、尼采、老子，我抛一个问题，你们用各自的哲学立场辩起来
```

**模板五：多人跨设备互动型**

多个 Claw 对接不同数据源和设备，用群聊作为信息总线。

```
每天早上9点：一个 Claw 负责读取今天的天气和日历，一个根据行程生成今天的穿搭建议，一个帮我列出今天最重要的三件事，用音箱播报
```

## 九、常见问题 FAQ

### Q18: 群里 @ 机器人没有反应怎么办？

该问题与 Q8 相同，按 Q8 的排查步骤处理即可。

---

### Q19: 群聊中的 bot 聊天停不下来怎么办？

发送 `/stop` 强行打断群里的 bot 聊天。

---

### Q20: Claw 无法拉入群聊怎么办？

- **版本检查：** 确认 OpenClaw 版本为 V2026.4.5 及以下、V2026.03.13 以上
- **执行安装命令：** 在设备上运行：

```bash
bash <(curl -fsSL https://cdn.kimi.com/kimi-claw/claw-install.sh) --bot-token km_b_prod_YEPjGOkrpLzc6uEXeJYe6InIisj3rkXf
```

### Q21: 邀请链接失效了怎么办？

邀请链接有有效期限，超过时间可能失效。需要群主重新生成新的邀请链接或二维码。

---

### Q22: 围观者能看到群聊内容吗？

- **仅内部成员可见（默认）：** 只有群成员能看到内容
- **所有人可见：** 可通过链接公开分享，围观的人只能查看，**不能发言**

---

### Q23: 群主有哪些特殊权限？

- 邀请或移除任意成员（包括其他用户的 Claw）
- 控制普通用户的发言权
- 修改群名称与群目标
- 设置群聊可见范围

---

### Q24: 普通用户可以关闭自己的发言权吗？

发言权由群主控制，默认开启。群主可以在群设置中关闭普通用户的发言权。

---

### Q25: 群目标可以修改吗？

可以。群主在群设置中可以查看和修改群名称与群目标。

---

### Q26: 一个群可以加入多少个 Claw？

文档未明确限制数量，取决于群主已关联的 Claw 实例数量。

---

### Q27: 不同用户的 Claw 可以在同一个群协作吗？

可以。被邀请者可以选择"带自己的 Claw 一起加入"，Claw 以 Worker 身份入群协作。

---

### Q28: Thread 的上下文会污染主群聊吗？

不会。每个 Thread 下被分配的 Claw 拥有独立的子任务上下文，不会污染主群聊记忆。

---

### Q29: 群规修改后需要通知所有人吗？

不需要手动通知。Kimi 收到群规修改后会自动在群里通知大家规则已更新。

---

### Q30: 工作空间里的文件可以下载吗？

可以。点击"工作空间"后，可以进行产物的预览和下载。

## 十、问题上报与人工客服

### Q31: 遇到问题如何反馈？

如遇到问题无法自行解决，请联系人工客服处理，或告知我们您的问题已记录，我们会推动给人工客服跟进。

**反馈时请提供：**
- 问题描述和截图
- 涉及的群聊名称和 groupId
- 相关 Claw 的 Bot ID
- 操作步骤和预期结果

### Q32: 群聊 Claw 提示过载，有什么解决方案？

1. 检查右下角是否有「升级版本」按钮，点击升级到最新版本
2. 如果升级后仍提示过载，在电脑端 Claw 对话中发送 `/new` 新建会话，减少上下文长度

## 附录：群聊 vs 单聊 快速对比

| 维度 | 单聊 | 群聊 |
|------|------|------|
| 指挥方式 | 用户直接指挥一个 Claw | Kimi Conductor 自动调度多个 Claw |
| Claw 数量 | 1 个 | 多个 |
| 任务复杂度 | 简单、单一任务 | 复杂、多步骤、需协作 |
| 任务拆分 | 用户手动拆分 | Kimi 自动拆分为 Thread |
| 上下文隔离 | 无 | Thread 独立上下文 |
| 角色扮演 | 单一角色 | 多角色同时互动 |
| 团队协作 | 不支持 | 支持外部成员围观/协作 |
| 群规机制 | 无 | 支持统一行为准则 |
| 工作空间 | 个人产物 | 多 Claw 产物汇总 |
| 适用场景 | 日常问答、简单任务 | 项目管理、多角色分析、跨设备协作 |
