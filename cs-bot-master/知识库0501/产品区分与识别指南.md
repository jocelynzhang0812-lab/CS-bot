# Kimi 产品区分与识别指南

> **必读：** 本指南用于防止将 **Kimi Code CLI**（终端命令行）与 **Kimi Claw / OpenClaw**（IM 机器人）混为一体。两个问题体系完全不同，配置项、命令、文件路径均不通用。**答错产品 = 答错答案。**

---

## 一、一句话区分

| 问题 | Kimi Code CLI | Kimi Claw / OpenClaw |
|------|---------------|----------------------|
| **我在哪里用它？** | 本地终端（Mac Terminal、Windows CMD/PowerShell、WSL、iTerm） | 微信、飞书、企微、微博、钉钉 |
| **我怎样启动它？** | 敲 `kimi` 进入交互式终端 | 在 IM 里 @机器人或直接发消息 |
| **它的配置文件在哪？** | `~/.kimi/config.toml` | `~/.openclaw/openclaw.json` |
| **它关推理用什么命令？** | `show_thinking_stream = false`（config.toml）或 `--no-thinking` | `/reasoning off`（聊天命令） |
| **它的会员/计费体系？** | Coding Plan，通过 kimi.com/code 管理 | Allegretto 及以上会员，通过 kimi.com/bot 管理 |
| **它的核心用途？** | 编程辅助、代码编辑、项目理解、Shell 操作 | IM 消息收发、群聊协作、自动化任务 |

## 二、高频混淆场景速查

### 场景 1：用户说"怎么关 thinking / reasoning"

**必须先问一句：你是在终端里用，还是在微信/飞书里用？**

| 用户实际场景 | 正确回答 | 错误回答（张冠李戴） |
|-------------|---------|-------------------|
| 终端里 `kimi` 运行时 thinking 废话多 | 改 `~/.kimi/config.toml`：`show_thinking_stream = false` | `/reasoning off` ❌ |
| 飞书/微信里 Kimi Claw 发一大段推理 | 发 `/reasoning off` | `show_thinking_stream = false` ❌ |

### 场景 2：用户提到配置文件

| 用户提到的文件 | 对应产品 | 绝不能答成 |
|--------------|---------|-----------|
| `~/.kimi/config.toml` | **Kimi Code CLI** | Kimi Claw |
| `~/.openclaw/openclaw.json` | **Kimi Claw / OpenClaw** | Kimi Code CLI |
| `AGENTS.md`（项目根目录） | **Kimi Code CLI** | Kimi Claw |

### 场景 3：用户提到命令或参数

| 用户提到的命令 | 对应产品 | 说明 |
|--------------|---------|------|
| `/reasoning off` / `/reasoning on` | **Kimi Claw** | IM 聊天斜杠命令 |
| `/login`, `/init`, `/compact` | **Kimi Code CLI** | 终端交互斜杠命令 |
| `--no-thinking`, `--thinking` | **Kimi Code CLI** | CLI 启动参数 |
| `--thinking off`（openclaw agent） | **OpenClaw** | Agent 运行参数，与 CLI 不同 |
| `kimi web`, `kimi acp` | **Kimi Code CLI** | CLI 子命令 |

### 场景 4：用户提到 IDE / 编辑器

| 用户提到的工具 | 对应产品 | 说明 |
|--------------|---------|------|
| VS Code + Kimi Code 扩展 | **Kimi Code CLI 生态** | 通过 ACP 协议集成 |
| Zed + Kimi Code | **Kimi Code CLI 生态** | |
| JetBrains + Roo Code + Kimi | **Kimi Code CLI 生态** | 通过 API Key 接入 |
| 微信/飞书里的机器人 | **Kimi Claw** | 与 CLI 完全无关 |

## 四、回答前自检清单（SOP）

在回答任何涉及"Kimi"的编程/配置/推理问题前，必须完成以下检查：

1. **用户提到了哪个配置文件？**
   - `~/.kimi/config.toml` → CLI
   - `~/.openclaw/openclaw.json` → Claw

2. **用户在哪个界面操作？**
   - 终端 / IDE → CLI
   - 微信 / 飞书 / 企微 → Claw

3. **用户用了什么命令格式？**
   - 以 `kimi ` 开头的命令 → CLI
   - 以 `/` 开头且在 IM 里 → 看上下文，可能是 Claw 的 `/reasoning`
   - `--` 开头的参数 → CLI 或 OpenClaw agent，需进一步确认

4. **如果仍不确定，必须反问用户：**
   > "您是在终端里敲 `kimi` 命令使用，还是在微信/飞书等聊天软件里使用 Kimi Claw？"

## 五、错误示例与纠正

### 错误示例 1

> **用户问：** "怎么关闭 kimi 的推理过程，废话太多了"
>
> **错误回答：** "在 `~/.kimi/config.toml` 里设置 `show_thinking_stream = false`"
>
> **问题：** 用户如果是在飞书里用 Kimi Claw，`show_thinking_stream` 根本不生效。
>
> **正确做法：** 先确认产品。若用户无法确认，给出分支回答：
> - 终端用户 → `show_thinking_stream = false`
> - IM 用户 → `/reasoning off`

### 错误示例 2

> **用户问：** "我的 claw 不回消息了，怎么修复配置？"
>
> **错误回答：** "运行 `kimi doctor` 或检查 `~/.kimi/config.toml`"
>
> **问题：** Kimi Claw 的修复配置在 OpenClaw 设置里，和 CLI 完全无关。
>
> **正确做法：** 引导用户到 Kimi Claw 设置页点击【修复 Kimi Claw 配置】。
