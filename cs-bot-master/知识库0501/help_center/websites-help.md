# Kimi 网站功能介绍

## Kimi Websites 网站是什么

Kimi Websites 是 Kimi Agent 的全栈建站产品，由 Kimi K2.5 多模态视觉代理 AI 模型驱动。核心能力是可视化编程（Vibe Coding）——用自然语言、设计图或录屏即可生成完整的全栈网站，并支持在线预览与一键发布。

Kimi Websites 生成的网站涵盖：

- **前端**：HTML 页面结构、CSS 样式布局、JavaScript 交互逻辑（动效、动画等）
- **后端**：数据库持久化存储、用户登录与认证
- **工程化**：自动版本管理、全栈预览、公网部署

### 产品功能说明

| 功能           | 说明                                                       |
|----------------|------------------------------------------------------------|
| 对话式建站     | 用自然语言描述需求，AI 自动生成全栈网站                     |
| 图片/视频理解  | 上传设计图/截图/录屏，AI 自动解析并生成页面                 |
| 持久化存储     | 自动构建后端数据库，支持数据的持久化读写与管理               |
| 数据库管理     | 提供可视化数据库面板，支持查看、编辑、删除等基础数据操作     |
| 用户登录       | 支持 Kimi 账号登录，也可通过数据库搭建账号密码登录系统       |
| 版本管理       | AI 自动进行版本提交（git commit），支持回退到任意历史版本    |
| 全栈预览       | 支持前后端完整预览，部分场景需手动启动沙盒环境               |
| 多轮修改       | 支持多轮对话迭代修改，查看不同版本效果                       |
| 发布与部署     | 手动点击「发布」后部署到公网，未发布前仅限预览，支持全栈部署  |
| 模板选择       | 提供多种预设模板，快速开始建站                               |
| 代码导出       | 支持打包下载完整项目文件（含前后端），自主部署               |

## 如何使用

### 方法一：网页访问
访问 Kimi Websites 专属页面

1. 打开浏览器：使用 Chrome、Safari、Edge、Firefox 等主流浏览器，
2. 打开官方访问地址：Kimi Websites 专属页面：[https://www.kimi.com/zh/websites](https://www.kimi.com/zh/websites)（支持中文/英文两种界面）
3. 登录账号：使用 Kimi 账号登录（支持手机号、邮箱注册）
4. 输入建站任务，Kimi Websites 提供三种主要建站方式：
   1. **文字输入**：在对话框中输入任务，用自然语言描述想要的网站效果；
   2. **图片/网站复刻**：通过上传设计稿/喜欢网站的截图触发对应功能；
   3. **模板建站**：快速搭建标准网站，选择预设模板，修改内容即可；
5. 开始建站：进入 Websites 功能界面，选择模板或直接对话创建。

### 方法二：移动 App 访问
移动 App 选择专属网站工具

iOS/Android/鸿蒙 均有官方 App（应用名“Kimi”），在输入框上方，点击工具栏（Taskbar）→ 切换至网站（Websites）Agent 模式，内含多模态生成工具与 Agent 功能，可进行视觉输入并生成前端代码。

### 方法三：通用 Agent模式
在通用 Agent 模式下，输入建站任务需求

在官网或移动设备，切换到 K2.5 Agent 模式，并输入建立网站相关的任务指令，Kimi 会调用视觉编程能力和代码执行工具。

### 实际案例演示

### 建站流程概览

Kimi Websites 会激活 webapp-building SKILL，这是一个专门用于网站构建的多轮代码生成代理，用于自主执行流程：

1. 需求解析：自动分析网站功能模块、视觉风格、交互逻辑
2. 任务规划：自主拆解为设计、开发、素材准备等子任务
3. 技术方案：自动选择适配的技术架构（如 React + Tailwind CSS）
4. 素材生成：按需生成图片、图标等视觉素材
5. 代码构建：编写页面结构、样式与交互代码
6. 多轮优化：根据预览效果自主调整，直至完成

### 预览与调整

- **实时预览**：生成 HTML 演示地址，可立即在浏览器中查看效果
- **对话修改**：继续对话要求调整，如“把导航栏改成深色”

#### 真实案例

### 可视化编辑

- **多轮优化**：通过持续对话精细调整网站细节
- **修改方式**：
  - **标注模式**：在网页上，点击需要修改的地方，用自然语言添加批注
  - **选择模式**：可以选择网页上的任意元素，并添加批注
  - 通过矩形、箭头、画笔、切换颜色等多种选择框，修改工具，添加修改建议
- 在多处完成批注后，将批注添加到对话，一次性给 Kimi 发出修改建议

### 代码部署与发布
| 部署方式   | 说明                                                 |
|------------|------------------------------------------------------|
| 自动部署   | Kimi 提供一键部署服务，生成可访问的在线链接          |
| 下载部署   | 打包下载代码文件，自行部署到服务器                  |
| 绑定域名   | 将生成的网站绑定到域名                              |

#### 分版本预览
生成完毕后进入预览页面，
可在预览界面，点击不同部署版本，下载不同版本文件。

#### 分享
点击分享按钮，可以获得公开部署的链接，并进行复制分享。也可以选择设置为私密可见。

### 多轮修改 

如果局部不满意，在预览窗口点击编辑按钮，继续修改。
也可以用自然语言发送给Kimi，继续用Agent进行修改。
确认效果后下载完整的 HTML/CSS/JS 代码，可直接部署或继续用编辑器精修。

# Kimi Websites 全栈能力说明
## 网站全栈能力升级

一个完整的网站通常包含**前端界面、后端服务和数据存储**三部分。Kimi Websites 已升级为全栈建站能力，支持从前端页面到后端数据库的完整网站生成与部署。

| 能力         | 说明                                                           |
|--------------|----------------------------------------------------------------|
| 前端生成     | HTML/CSS/JavaScript 页面结构、样式与交互逻辑                    |
| 后端数据库   | 自动构建后端数据库，支持数据持久化读写与可视化管理               |
| 用户登录     | 支持 Kimi 账号登录，也可通过数据库搭建账号密码登录系统           |
| 版本管理     | AI 自动进行版本提交，支持回退到任意历史版本                      |
| 全栈预览     | 支持前后端完整预览，部分场景需手动启动沙盒环境                   |
| 发布与部署   | 手动点击「发布」后部署到公网，未发布前仅限预览，支持全栈内容部署  |

## Agent 的任务交付物

- 完整的任务执行文件（含前后端代码）
- 可在线预览和发布的全栈网站
- 可下载的 Zip 包（位于 `/mnt/okcomputer/output/` 目录）

## 当前限制

以下场景暂不支持，后续版本将陆续开放：

- 第三方支付接口对接（如微信支付、支付宝）
- 第三方 OAuth 登录（如微信、GitHub，当前仅支持 Kimi 账号和账号密码登录）
- 复杂的服务端 API 集成（如对接外部 SaaS 服务）
