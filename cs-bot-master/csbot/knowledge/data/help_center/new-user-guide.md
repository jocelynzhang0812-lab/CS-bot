你可以在对话框中输入问题、上传文件，或根据需要切换不同模型。

## 默认搭载 K2.5 模型

Kimi K2.5 开源模型是 Kimi 迄今**最智能**的模型，在 Agent、代码、图像、视频及一系列通用智能任务上达到开源领域最先进水平（state-of-the-art）。Kimi K2.5 也是 Kimi 迄今**最全能**的模型，原生的多模态架构设计，同时支持视觉与文本输入、思考与非思考模式、对话与 Agent 任务。

因此，Kimi 除了擅长问答对话，还能够自主规划任务、调用工具并完成多步推理，支持复杂任务的深度思考与执行。

### K2.5 内置官方工具

无需额外配置即可使用：

| 工具 | 功能说明 |
|------|----------|
| Web Search | 实时联网搜索，获取最新资讯与动态 |
| Fetch Tool | 定向访问指定 URL（网页、文档、图片等在线资源） |
| Data Source | 连接专业数据库，支持查询数据库列表及获取数据资源 |
| Image Search | 以文搜图、以图搜图，快速定位视觉内容 |
| IPython | 执行 Python 代码，进行数据与可视化分析 |
| Memory | 智能记忆功能，记录你的个人习惯与偏好设置 |

## 视觉与多模态能力

K2.5 支持图片、视频、PDF/Word/Excel 等办公文档的全场景理解，采用**原生多模态架构**（非外挂 OCR），可直接解析文档内容并执行中高阶办公操作（如编写 Excel 公式、生成 PPT 大纲/VBA 代码）。

### 多模态输入支持

- **图片**：PNG、JPEG、WebP、GIF 等（支持 URL、Base64、本地文件）
- **视频**：MP4 等（支持 video_url 或图像序列帧 + fps 参数）
- **文档**：PDF、Word、Excel、TXT、Markdown 等
- **在线资源**：URL（统一资源定位符）、公众号文章链接
- **以图搜图**：上传图片，调用图像检索工具搜索相关或相似图片
- **视频理解**：K2.5 首次引入视频输入能力，可分析录屏内容并复现交互逻辑
- **视觉编程**：支持上传截图或录屏生成前端代码（HTML/Vue/React 等），可基于设计稿自动还原网页

### 会话输出格式

- 纯文本 / 图文混排：常规对话回复
- Markdown：富文本、标题、列表、表格、链接
- 代码块：HTML/Vue/React/Python/JSON/Mermaid 等（带语言标识）
- JSON 结构化数据：强制 JSON 模式输出
- Mermaid 图表：流程图、时序图、思维导图等语法
- LaTeX 数学公式：行内和块级数学表达式
- 双模式输出（Thinking）：同时返回 `reasoning_content`（思考过程）和 `content`（最终答案）
- 工具调用格式：Function Calling 标准格式（面向开发者）

### 注意事项

1. **生图限制**：标准会话模式下仅支持搜索图片，不支持直接生成图像（生图功能需使用 Agent 模式）
2. **文件输出限制**：仅支持输出文本内容（如 PPT 大纲、Word 正文、代码等），不支持直接输出二进制文件（如 `.pptx`、`.docx`、`.xlsx`、`.pdf`）
3. 如需使用生图功能或生成复杂可编辑文档，请使用 [Agent 模式](https://www.kimi.com/agent) 或配合代码执行工具。

## 会话 vs. 对话轮次

当你点击“新建会话”，想象你与 Kimi 进行一场长谈。

| 概念 | 通俗类比 | 技术定义 |
|------|----------|----------|
| 会话（Session） | 整场见面（从开始交谈到交谈结束） | 一次完整的聊天过程，有明确的开始和结束 |
| 对话轮次（Turn） | 一轮交锋（你说一句，Kimi 答一句） | 你的输入（Input）与 Kimi 的回复（Response）组成一个来回 |

### 关键区别

**会话（Session）**
- **记忆连续性**：Kimi 记得本轮聊天中的所有上下文
- **跨轮次关联**：每次回复都会参考前面的对话内容，作为新一轮对话的上下文
- **有始有终**：点击“新建会话”= 重新开始，历史上下文清空

**对话轮次（Turn）**
- **累计计数**：第 1 轮、第 2 轮……逐轮累加
- **记忆有限**：轮次过多（如超过 50 轮）可能导致“遗忘”早期内容，尤其是长篇小说创作、有明确限制和要求的场景
- **受限于上下文长度**：受 Token（字数）限制，轮次过多时最早的内容可能被压缩

## 为什么需要区分这两个概念

在同一个会话中，每次你输入了新的问题/指令，Kimi 当前轮次回复都会参考前面的对话轮次。当对话轮次过多、内容积累过长时，回答质量可能会有所下降。

### 典型场景

**场景 1：开始新话题**

**场景 2：长文档处理**
- 当你在一个会话里聊了几十轮后，AI 可能“遗忘”最开始设定的规则或上传的文件摘要
- 对策：重要任务中途主动总结（“总结一下我们目前的方案”），或开启新会话

**场景 3：理解产品限制**
- 上下文长度限制通常按 Token（字数）计算，但通俗理解就是“轮次”
- 如果看到“新开会话”提示，意味着当前会话轮次太多，需要新开会话

## 使用建议

1. **每个任务单独新建会话**：每个独立任务，如写论文、查资料、编程调试等，各自新建会话，避免上下文相互干扰
2. **长会话定期总结**：超过 20 轮对话后，主动让 AI 总结关键信息，防止信息丢失
3. **敏感信息及时清理**：会话历史会保留，涉及隐私的内容完成后可删除整个会话

# Kimi 记忆空间介绍

## 什么是记忆？
Kimi现在真正学会了倾听，能做到越交谈，越默契。具体来说：
Kimi不再只回应对话框的当下，在与用户的每一次对话中，记下重要的细节——你的偏好、习惯、风格...
随着使用的增加，Kimi在跨会话中持续学习，不断调整自己理解你的方式
用户不需要重复诉求，Kimi会主动考虑“我们聊过什么”，据此激发出更贴切，更连贯的理想回应

## 什么会被记住？
Kimi单独训练了一个模型，让Kimi有针对性地记住能为你提供价值的会话内容。
在记录前，Kimi会先问自己两个问题，决定记不记：
- 面前的这位用户是谁？
- 这位用户在意什么、偏好什么、雷点是什么？

## 如何触发记忆
记忆已经成为kimi内化的一个能力项，自然对话即可触发记忆能力。
更新或管理Kimi记忆功能的方法：

（1）对话中直接指令（最常用）
你可以直接通过自然语言指令让Kimi更新记忆：

| 操作 | 你可以说的话 | Kimi的响应 |
|:---|:---|:---|
| 添加记忆 | “记住我喜欢喝咖啡”<br>“记下我的工号是 9527”<br>“以后记住我是项目经理” | Kimi会调用记忆工具保存 |
| 修改记忆 | “更正一下，我搬到上海了”<br>“更新：我现在是高级工程师” | Kimi会替换旧记忆 |
| 删除记忆 | “忘掉我喜欢咖啡”<br>“删除关于我工号的记忆”<br>“清空所有记忆” | Kimi会移除指定或全部记忆 |

（2）设置中管理（界面操作）

根据系统提示，你也可以在设置中自行管理：
- 网页版/手机 App: 设置 → 个性化 → 记忆空间
- 在这里可以查看所有已保存的记忆条目
- 可以手动删除不需要的记忆

### 新增记忆
对话中会自动触发，比如：
“我叫xxx，朋友叫我xxxx”
“我是小学数学老师，所以课件需要考虑到小朋友的受众”
“你以后千万别给我发emoji了”
“我一般都用c++ code”
“以后说话不要说重复的车轱辘话，不要带标点符号”

如果有什么特别想让kimi记住的，可以单独强调一下：
“记住.....”
“请你记得....”
“.......，不要忘了”

### 更新记忆
可以在对话中新增记忆后，删除不需要的单条记忆
可以在用户主页-设置-个性化-记忆空间，点击删除单条记忆

### 删除（忘记）记忆
可以在用户主页-设置-个性化-记忆空间，点击删除单条记忆

### 查看记忆
直接问：
“你当前都记住了我什么？”

# Kimi 概述

Kimi 是由 Moonshot 自主研发的 AI 智能助手，支持联网搜索、深度思考、多模态推理和超长文本对话。

访问 [Kimi.com](https://www.kimi.com/) 或下载 Kimi App，即可开始聊天、创作、研究和构建应用。开发者可前往 [Kimi 开放平台](https://platform.kimi.ai/) 获取 API 与工具，将 Kimi 能力集成到自己的应用中。

## 会话模式

- **联网搜索**：连接网络获取实时信息，支持搜索网络信息。
- **快速模式**：快速回答相关信息。
- **思考模式**：支持更深入的多轮思考与搜索——非常适合编码、逻辑或分析任务。

## Agent 智能代理

Kimi 不仅是对话助手，更是能自主执行任务的 AI Agent：

- **[通用 Agent](https://www.kimi.com/agent)**：自动规划并完成任务，支持网站生成、PPT 制作、深度研究、文档和表格处理等。
- **[Agent 集群](https://www.kimi.com/agent-swarm)**：支持高达 1500 次并行工具调用，可以自主调度多达 100+ 子智能体（Sub-agents）并行处理任务，适用海量搜索、长文写作、批量处理任务。
- **[Kimi Code](https://www.kimi.com/code?from=kfc_overview_topbar)**：面向开发者的编程助手套件，包括命令行工具（CLI）、VS Code 插件。
- **[Kimi Claw](https://www.kimi.com/bot)**：零部署云端自动化平台，无需服务器或 Docker，30 秒内即可启动持续运行的 AI 代理。内置 5,000+ 技能库（ClawHub），支持链式组合调用与多步骤自主规划，轻松完成复杂调研与数据分析流程。

## 其他核心功能

- **特色功能**：拍照解题、语音通话、翻译、写作。
- **文件处理**：支持 PDF、Word、Excel、PPT、图片、TXT、视频（单个不超过 100M），最多支持 50 个文件。
- **常用语**：添加你常用的快速提示，适合把高质量、可复用的提示词设置为常用语。

# 常用语介绍

常用语是 Kimi 提供的快捷提示功能，让你可以将高频使用的提示词保存下来，在任何会话中一键调用，省去重复输入的麻烦。

## 适合场景

- **固定格式的任务**：如每次都需要 Kimi 按特定模板输出会议纪要、周报、邮件等
- **高质量提示词复用**：精心调试好的提示词，保存后可反复使用
- **常用指令快捷触发**：如“总结这篇文章”“翻译为英文”“用表格整理”等日常高频指令

## 如何添加常用语

1. 打开 [kimi.com](https://www.kimi.com/) 或 Kimi App
2. 点击对话输入框左侧的 **「+」** 按钮，或找到**常用语入口**
3. 点击 **「添加常用语」**
4. 输入常用语标题（方便识别）和具体内容（完整的提示词文本）
5. 保存即可

## 如何使用常用语

在任意会话的输入框中，点击常用语入口，选择已保存的常用语，内容会自动填入输入框。你可以直接发送，也可以在此基础上补充修改后再发送。

## 管理常用语

- **编辑**：点击已有的常用语条目进行修改
- **删除**：移除不再需要的常用语
- **排序**：将最常用的常用语排在前面，方便快速选取

## 使用建议

- 将经过反复验证的优质提示词设为常用语，确保每次都能获得稳定、高质量的输出
- 为不同工作场景（写作、分析、翻译等）各准备几条常用语，快速切换工作模式
- 常用语支持长文本，复杂的多步骤指令也可以保存

# Kimi Agentic 搜索介绍

Kimi 的 Agentic 搜索采用端到端自主强化学习（End-to-end Agentic RL）架构，区别于传统的工具编排方案，让 AI 自主决定何时搜索、调用何种工具、如何修正策略，从而完成复杂的信息搜集与处理任务。
相比传统搜索的关键词匹配，Kimi 的搜索能力经历了两次核心进化：
- 探索版搜索（2024年10月）实现了大模型语义理解 + 实时信息检索，可自动拆解复杂问题、溯源信源；
- Agentic 搜索则进一步实现了自主规划与工具调用，支持金融数据查询、学术文献检索、图片搜索等多模态任务。

## 搜索联网设置

联网搜索用于突破知识时效边界，获取即时、可信、可溯源的最新信息。
点击对话输入框下方的「联网搜索」开关（显示为🌐图标）。开启后，Kimi 将自主判断当前问题是否需要实时数据，智能调用搜索引擎与垂直数据库完成信息检索与整合。
## 核心能力

1. **即时性与权威性并重**
实时检索互联网信息，覆盖 100+ 经过验证的可信来源（主流新闻媒体、政府公告、金融数据平台、学术期刊库），确保资讯既新鲜又权威。

2. **信息溯源机制**

# 提示词入门指南

## 什么是提示词（Prompt）？

提示词是你发送给 Kimi 的文字指令——可以是一个问题、一段描述、一组要求，甚至是一份完整的任务说明。Kimi 根据你的提示词来理解意图、组织回答并执行任务。

简单来说：**你怎么问，决定了 Kimi 怎么答。** 一个清晰、具体的提示词，往往能带来更精准、更有价值的回复。

## 好提示词的四个核心要素

### 1. 明确角色与背景

告诉 Kimi 你是谁、在什么场景下提问，能帮助它更好地匹配回答的深度和风格。

### 2. 具体描述任务

越具体的描述，越能减少来回修改的次数。好的任务描述通常包含：

- **做什么**：明确的动作（分析、总结、翻译、生成、对比……）
- **输出格式**：表格、列表、段落、代码、Markdown……
- **数量与范围**：字数、条目数、时间范围、地域范围……
- **质量要求**：学术风格、口语化、简洁、详尽……

## 实用技巧

### 善用追问和迭代

第一次回答不够满意？直接追问调整，无需重新描述所有背景：

- “请更详细地展开第 2 点”
- “语气改得更正式一些”
- “请补充数据来源”
- “换一个角度重新分析”

### 善用文件和链接

Kimi 支持上传 PDF、Word、Excel、图片等文件，也支持直接粘贴 URL。结合文件提问能大幅提升效率：

- “总结这份 PDF 的核心内容”
- “根据这份 Excel 数据，分析销售趋势”
- “阅读这个链接的文章，提取关键信息”

### 指定输出格式

明确告诉 Kimi 你想要的输出格式：

| 需求 | 提示词示例 |
|-----|-----------|
| 表格对比 | “请用表格对比 A 和 B 的优劣势” |
| 代码输出 | “请用 Python 实现，包含注释” |
| 结构化列表 | “请用编号列表整理，每条不超过 20 字” |
| Markdown | “请用 Markdown 格式输出，包含标题层级” |

### 利用系统提示和记忆

如果你有固定的偏好和要求，可以通过 Kimi 的**记忆功能**让 Kimi 长期记住：

- “记住我是一名前端工程师，偏好 React 技术栈”
- “以后回复都使用简体中文，用 Markdown 格式”
- “回答技术问题时请附上代码示例”

这样你不用每次都重复说明，Kimi 会在后续对话中自动参考这些偏好。

## 常见误区

| 误区 | 正确做法 |
|-----|---------|
| 一次塞入过多不相关的任务 | 每个独立任务新建会话，保持上下文清晰 |
| 提示过于模糊（如“帮我写点东西”） | 说明写什么、给谁看、什么风格、多少字 |
| 期望一次完美输出 | 先生成初稿，再通过追问迭代优化 |
| 不检查 AI 输出的事实性 | 涉及数据、日期等事实信息时，务必核实 |
