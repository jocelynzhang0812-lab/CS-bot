# kb_04 — 错误码速查表

## 1. 错误码速查表

### 1.1 HTTP 错误码与处理方法

# 1.1 HTTP 错误码与处理方法

| 状态码 | 错误类型 | 含义 | 处理方法 |
|--------|----------|------|----------|
| 400 | content_filter | 内容审查拒绝，输入或生成内容包含不安全/敏感内容 | 避免输入敏感提示语；发送 /new 新建会话或 /reset 重置 |
| 400 | invalid_request_error - Input token too long | 请求 token 超过模型最大限制（通常由 session 膨胀或 MEMORY.md 过大导致） | 发送 /new 创建新对话；若仍超限，让 bot 清理 MEMORY.md 或工作空间文件 |
| 400 | invalid_request_error - Token limit exceeded | token 数与 max_tokens 之和超过模型规格 | 发送 /new 创建新对话；检查是否有 Skill 误加载过多上下文 |
| 400 | invalid_request_error - File too large | 上传文件超过 100MB | 压缩后重新上传 |
| 401 | invalid_authentication_error | API Key 失效或过期 | 设置 → 【修复 Kimi Claw 配置】；若修复后仍 401 → 【恢复初始设置】；还不行则删除重建 |
| 401 | incorrect_api_key_error | API Key 未提供或不正确 | 同上一行 |
| 401 | auth_failed (Bridge/WebSocket) | 平台侧认证失败，可能伴随 will not retry | 设置 → 【修复 Kimi Claw 配置】；若提示"无法验证会员权益" → 见下方会员过期 |
| 403 | permission_denied_error - API not open | 访问的 API 暂未开放 | 设置 → 【修复 Kimi Claw 配置】 |
| 403 | permission_denied_error - Model not allowed | 模型不在平台白名单（如 K2.6 kimi-for-coding 在 im_only 下可能被拒） | 切换回 K2.5 (k2p5)，等待平台更新白名单 |
| 404 | resource_not_found_error | 模型不存在或无权访问 | 检查模型 ID；若刚升级过 OpenClaw，可能是插件兼容性问题 |

# 1.2 平台/Bridge 层错误（非 HTTP，用户可直接看到）

| 用户看到的报错 | 底层错误信息 | 含义 | 处理方法 |
|---------------|-------------|------|----------|
| "LLM请求被拒绝：我们目前无法验证您的会员权益" | resource_exhausted / membership_validation_failed | Allegretto 会员已过期，平台拒绝所有 LLM 请求 | 本地任何操作无效。检查 Kimi 账号会员状态：若过期则续费；若正常则删除重建 bot（平台权益同步延迟） |
| "Bot 未连接" 或 "连接失败" | bridge ACP disconnected / subscribe failed reason=stream_error | Bridge WebSocket 断开，Gateway 与平台失去连接 | 设置 → 重启 Gateway；若仍失败 → 【修复 Kimi Claw 配置】 |
| "与云端电脑的连接已断开" | failed to execute script via SSH and cloud assistant | 远程 VM 已离线或崩溃 | 【修复配置】和【恢复初始设置】都失败 → VM 完全挂了，只能删除重建 |
| "服务繁忙，请稍后再试/The engine is currently overloaded, please try again later 当前请求过多，请稍后重试" | code=resource_exhausted message=connect stream failed | 平台侧资源耗尽（K2.5 节点过载） | 等待后重试；发送 /new 减少上下文。不是用户 tier 问题 |
| bot 初始化失败，反复重启 | Config invalid / plugins.entries 包含已删除插件 | Stale plugin 残留导致配置验证失败 | 【恢复初始设置】；或联系技术侧清理 plugins.allow 和 plugins.entries |
| 所有 edit/记忆操作失败 | Edit: in ~/.openclaw/workspace/AGENTS.md (xxxxx chars) failed | thinkingDefault: "high" + K2.5 im_only 导致 tool call 参数错误；或文件 >10KB | 设置 → thinking 改为 "low" → 重启 Gateway；若仍失败 → /new 后让 bot 清空 AGENTS.md 重新初始化 |
| 消息发送失败 | message om_xxx expired, discarding | 消息在队列中滞留过久被丢弃 | 检查是否有大量消息堆积；重启 Gateway；清理 failed queue |
| 升级后飞书/微信不可用 | openclaw-weixin@1.0.3 / openclaw-lark@2026.3.18 兼容错误 | Auto-upgrade 后插件与新版本 OpenClaw 不兼容 | 等待官方适配；或【恢复初始设置】回退到兼容版本 |
| "服务繁忙，请稍后再试" 或固定回复 `⚠️ API rate limit reached` | `code=resource_exhausted message=connect stream failed` / `engine_overloaded_error` | **Kimi 平台侧 K2.5 节点容量不足**，非用户账号问题 | 等待几分钟后重试；发送 `/new` 减少上下文；频繁出现则反馈 Bot ID 给平台运维。**不需要充值升级 tier** |
| "鉴权失败，API Key 不正确" / "Bot 状态异常" | HTTP 401 `invalid_authentication_error` / `incorrect_api_key_error` | API Key 失效或过期 | 设置 → 【修复 Kimi Claw 配置】；若修复后仍 401 → 【恢复初始设置】；还不行则删除重建 |
